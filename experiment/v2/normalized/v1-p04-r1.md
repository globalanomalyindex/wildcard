## angles

- Full document deserialization tokenizes and types all fields whether used or not, crossing expensive bonds -> The scorer likely reads only a subset of fields -> Consider a streaming partial parser that exits early after extracting only required fields; worth profiling which fields are actually read before treating deserialization as an irreducible cost.

- A block of mica stops light entirely, but a thin lamina transmits with fidelity; the same substance becomes accessible only when thickness is reduced -> Fraud signal is not uniformly distributed across 20M messages; high-risk transactions cluster in specific segments identifiable from metadata before full parse -> Use a priority lane triage strategy to identify and score the concentrated fraud-risk subset within SLA, while bulk backlog drains overnight; ask what the scorer would do if it could only process the right 5%.

- Optical interference in layered mica produces richer signal than any single layer; patterns emerge from aggregate -> Consumer loop processes single messages one-at-a-time through the same parse-and-score cycle, missing correlated patterns -> Micro-batch parsing amortizes setup cost and opens door to aggregate signals (e.g., three modest-risk transactions from the same device fingerprint in 200ms); same CPU budget may yield both more throughput and richer signal.

- Mica survived extreme furnace environments while remaining transparent; the measurement port existed inside the thing it was measuring -> Lag counter is produced by the same saturated consumer group infrastructure, and offset commit cadence degrades during peak load, understating backlog depth when measurement matters most -> Check whether lag alerting reads committed offsets (which lag behind true consumption when commit interval stretches) versus high-watermark tracking; a saturated consumer committing every 5s instead of every 500ms reports stale lag.
