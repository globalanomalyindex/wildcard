## angles
- Flowers appear ready but internal state has changed; translators work from a source state that is no longer current -> This is a stale-signal failure: the English page changed but translators' working copies didn't -> Does the English copy actively announce changes to downstream readers, or do they have to go looking? Passive distribution of a changed file looks identical to a stable one.
- Multiple workers reading the same signal calibrate differently without a shared reference, creating drift over time -> Three different German names for the same loyalty program represents calibration drift across independent translators, each anchored to their own history -> A locked glossary explicitly synced before each job concentrates the shared reference standard, like a mounted cross-section that all workers anchor to periodically.
- Expensive skilled judgment and laborious execution need not happen in the same person or at the same time; one worker marks ready flowers, others execute the transfer -> Change detection (which English segments are affected) is skilled judgment that either doesn't happen or happens late when complaints come in; translation execution is already distributed -> Concentrate the change-detection step—whether a weekly diff pass or automated flagging—leaving translation distributed, separating the marking step from the execution step.
