## angles
- The stone has a pre-existing rift plane (the terms causing customer confusion are a small, high-stakes, low-churn layer; marketing prose is a different, volatile stratum) -> this stratum structure is already set in the material, not being imposed -> differentiate the stratum: lock product names and UI labels across all translators per language, let promotional copy float freely; costs nothing, requires no extra review, targets the failure mode customers in Japan are hitting
- Each node must be pre-loaded together before force is applied (right now each freelancer acts as an independent node with no shared pre-load, causing cross-grain fracture) -> the structural fix is distributing the pre-stress across all nodes before anyone drives hard -> issue a per-language locked term sheet before any translator picks up a job; budget-neutral and turnaround-neutral, it changes the force geometry so all nodes are seated together
- A fracture's worst state is when it's already propagated to a customer-facing surface; the diagnostic is checkable before that happens (a term appearing in app UI, help article, or booking confirmation before a canonical form was locked across translators) -> "running wild" is detectable early by checking whether any node was loaded out of sequence -> wire the check into job assignment: has this term reached a customer-facing surface before a single canonical form was committed to?
