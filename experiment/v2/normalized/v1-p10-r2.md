## angles
- Velocity metric is oriented orthogonally to design-system consistency signal (like an intensity detector missing polarization state) -> the information exists but the measurement axis can't see it -> add a complementary metric alongside velocity that reads the axis it can't see (e.g., "components detached this sprint / total components touched")
- Drift is produced by the handoff surface between Figma and React (like polarization created by a reflective medium), not by either artifact alone -> the divergence happens at merge time when detached instances are rebuilt without corresponding props -> add a thin compensating layer at handoff that flags missing parity between production components and Figma library state
- Single metric gives one projection of state, not full state; design-system health is at least three-dimensional (component-coverage parity, behavioral-state coverage, handoff fidelity) -> velocity holds only one axis fixed while the others drift -> produce a three-number spanning summary per sprint that measures all three orthogonal dimensions
