## angles
- Operatic claque placement under architectural constraints (which cues fire where most matters) -> Documentation maintenance requires triage by consequence, not uniform effort -> Formalize a consequence inventory: score each help center page and README section on (1) change frequency and (2) damage if stale; apply hard review triggers only to top 5-10 high-consequence pages, relax cadence elsewhere
- Claque placement is pre-wired before the curtain rises; mid-act improvisation fails -> Learning about features at launch means responding after the moment of consequence has passed -> Add a required three-option PR field ("does this change affect user-facing docs?") and wire a selection to automatic pre-merge notification, placing documentation review in the change-propagation path architecturally
- Dead signals (applause after a performer leaves) drown out live ones -> Unreviewed Confluence pages with no owners rank above current content in search, creating false authority -> Export Confluence with last-modified dates; archive or banner all pages untouched since January 2024 with no owner; configure search to explicitly exclude archived pages rather than relying on algorithmic de-prioritization
