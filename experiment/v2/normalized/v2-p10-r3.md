## angles
- In wave-particle duality, complementary observables of the same thing are each fully valid within their own measurement context — attempting to force one apparatus to do both jobs destroys information. The Figma library and React package are two valid observables of the same UI intent, each optimized for different apparatus (design tool vs. runtime). -> Formally partition the 312 components into production-committed (React coverage, strict visual parity) and exploratory (design-phase only). Run the focus-state audit against production-committed tier only (~156 components) to create a defined surface for consistency commitments.
- A quantum system resolves into a definite state only at a measurement event. If the review protocol observes after shipping, inconsistencies are found only downstream when clients flag them. The current pipeline lacks a measurement event that forces resolution of interactive states (focus, hover, disabled) before production. -> Add a PR checklist requirement for production-committed components: a Storybook story covering focus, hover, and disabled states must exist and render before merge — a one-time policy change that makes observation constitutive of shipping rather than downstream.
- A one-time audit or quarterly review creates drift because the only crystallization event is the audit itself. What's needed is not a larger audit but a higher-frequency measurement schedule — small, regular events where one component is fully walked from Figma spec to Storybook story to production DOM. -> Block 30 minutes every Monday morning, pick one component from production-committed tier, and walk from Figma through Storybook to staging DOM, filing any gap as a specific ticket. The rhythm turns a closed audit into an open, accumulating record.
