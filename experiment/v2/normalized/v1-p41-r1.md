## angles
- Folder structure is evolutionarily lagged — calibrated to a 2019 workflow that no longer exists, making the old landmark cues unhelpful for navigation -> this is relict structure, not a guide to current function; treat it as geology to read -> stop trying to reconcile it and instead ask "what were the rules then?" to locate things like the wedding photos historically rather than functionally
- 80,000 photos at multiple resolutions with edited/original pairs represent accumulated genetic load with near-zero visible cost until storage hits capacity -> the threshold is crossed; load management doesn't require eliminating every variant, just reducing below threshold -> target high-cost, low-fitness variants first (resolution duplicates) before attempting full deduplication
- Any in-place reorganization (merge drives, fix folders, reconcile systems) is trapped on a local fitness peak that already proved inadequate -> escape requires changing the fitness landscape, not willpower on the existing climb -> build a new canonical archive first with no legacy structure, treat old drives as source material, and migrate only what can be confirmed into the new space
