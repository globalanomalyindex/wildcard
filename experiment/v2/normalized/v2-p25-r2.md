## angles
- Documentation stores (help center, README, Confluence) receive no signal when source changes; engineers ship and docs drift invisibly until customers encounter breakage -> the mirror failure mode is that the mirror never knows it's failed -> add a required PR template checkbox ("Does this change any user-facing behavior that is documented in the help center, README, or Confluence?") routed to a doc-review channel to catch drift at merge time rather than post-incident
- Help center, README, and Confluence have overlapping content with no declared canonical source; archived pages rank above live ones creating ghost images with no clear authority -> when a mirror has partial silvering, competing images appear simultaneously, neither flagged as authoritative -> assign each store an exclusive scope (help center owns customer-facing procedural content; README owns installation and local-dev setup only; Confluence owns internal process), then archive or redirect content outside scope mechanically, with search ranking following via 301 redirects
- Old articles that are still actively visited are the highest-decay-risk zone; they've drifted the most and are still being relied on -> decay begins at the perimeter and progresses invisibly while the surface looks fine -> export last-edited date and monthly page views for all help center articles, sort by age and traffic, and issue a validate-or-archive verdict on the top 20 results
