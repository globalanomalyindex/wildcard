## angles
- Tank space forces deliberate waves of work with explicit batch criteria based on value (genetic rarity, request frequency, at-risk status), not ease -> Archive triaging by resource constraints: 70,000 images but only 3 hours/week means a first-cycle batch based on likelihood of licensing request in next six months, leaving older material untouched until cycle two -> Define and complete one high-value batch at a time; leave the rest untouched until the next cycle; high-value material starts returning results immediately while scope stays bounded
- Provenance tags in coral micro-fragmentation are load-bearing: a fragment without a tag cannot be used for restoration and has zero commercial value, making it the first metadata field to attach -> Clearance/rights status is the load-bearing metadata: an image without a rights flag cannot be sold, and the lost licensing sale was lost because provenance wasn't attached, not because the image didn't exist -> Apply a simple three-value rights flag (cleared / needs review / uncleared) to every image in the first-cycle batch before any other metadata; a partially described archive with reliable rights flags is more useful for licensing than a fully described one with unknown clearance
- Single-axis retrieval systems amplify biomass without amplifying diversity and fail catastrophically when a different use case arrives; a coral nursery indexed only by genetics but growing all descendants from three colonies becomes a single failure point when a pathogen arrives -> The archive is indexed only by date (useful for production recall) but invisible to licensing queries (clearance status, performer names); when clearance became needed, the single-axis index collapsed the archive's value -> Build at least one additional retrieval axis in parallel with existing ones as new material is shot; rights status + date means the archive grows along two axes from here forward and the failure mode never compounds further
