## angles
- The translators are keying their glossaries to the English string surface rather than the underlying rift (the stone's latent grain) → when marketing edits the English name, each translator independently resolves what to call it, producing conflicting versions → assign each recurring term a stable internal ID token, store the canonical glossary against that ID instead of the English phrase, so the glossary remains resilient when English labels change without requiring new review rounds
- When the marketing team edits a live page, the wedge is driven all the way to publication before translators see it, concentrating stress → each translator independently resolves the new phrasing and diverges from the others → set up a lightweight delta-notification triggered only on source changes that posts excerpts of changed sentences to a shared translator channel, so a quick alignment pass can happen on the delta before individual delivery rather than after
- The app's UI renders a string while translated marketing copy invented a different label, creating a fracture → the app's string is the rift set by engineers and is not negotiable → export the app's own string table for each locale and load it into translation memory as a locked no-translate list, so translators see which strings must be used verbatim to match what customers see in the app
