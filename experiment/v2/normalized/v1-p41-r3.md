## angles
- Folder structure, filenames, and thumbnail previews are visually salient but don't capture what makes photos "the same" or "different" -> the diagnostic signal for photo identity is pixel-level perceptual hashing, not visual appearance or naming -> try content-hashing across all four sources as a single flat scan using tools like `fdupes -r`, `rdfind`, or `digiKam` to surface matching sets without navigating folder schemes
- Edited and original versions have opposite functions: one is the creative artifact, one is the lossless source -> treating them as duplicates to be collapsed loses the irreplaceable ground copy -> consider edited/original detection as a prior step before running deletion logic, ensuring the deduplication tool understands this relationship rather than collapsing the pair by apparent similarity
- When two files appear identical in thumbnail but differ in resolution or format, the tie-breaking rule is to examine diagnostic EXIF fields -> check OriginalFilename, Software, pixel dimensions, GPS, and flash data to determine which is closer to the camera original -> the file with more intact EXIF is almost certainly the one to keep in a tie
