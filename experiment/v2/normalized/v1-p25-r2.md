## angles
- Documentation loses coupling to the product after release — the help center auth guide persisted as wrong after engineering shipped a change, because no mechanism reconnects the document surfaces to the release pipeline. The suggestion: instrument the coupling with a reference beam from release to documentation surfaces (a lightweight signal when a change ships), before a customer's report becomes the detection mechanism.
- Documentation systems lack external verification of alignment with product state — the help center, README, and Confluence have no way to detect divergence; only a support ticket reveals the problem after fact. The structural need is an external comparator holding a known-good reference state (the release itself or its diff) that runs periodic comparison to check alignment.
- Search ranking treats age as neutral, surfacing archived pages above current ones — this is systematic distortion in the retrieval layer that presents as faithful results. The fix is not to recalibrate the documents themselves but to correct the optical path: separate "documentation is outdated" from "the search layer cannot distinguish current from archived," as they live in different places.
