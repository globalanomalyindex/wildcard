## angles
- An archive measured only on one coordinate (date) cannot be steered -> rights status, performer name, costume exist in real state but are invisible to the system -> add a minimum viable set of required fields at ingest going forward (production title, performer(s), costume/scene, rights clearance status), taking ~10 minutes per shoot to eliminate future bottlenecks on searchability and clearability
- Query load is concentrated in low-dimensional submanifold of state space, not uniformly distributed -> most future requests will hit 8-10 production titles repeatedly, not arbitrary dates or edge cases -> prioritize tagging the top 3,000-5,000 images from high-query shows using AI-assisted auto-tagging first, deferring the remaining 65,000 images without blocking high-value use
- Backlog is an integrating system with constant inflow and limited outflow -> at 3 hours/week processing rate with 6,000-7,500 new images annually, manual effort alone cannot discharge faster than intake enters -> run concrete arithmetic against actual shoot volumes to determine whether AI-assisted tagging can achieve discharge rate; if not, the budget constraint requires either an intern sprint or institutional subscription rather than continuous underpowered grinding
