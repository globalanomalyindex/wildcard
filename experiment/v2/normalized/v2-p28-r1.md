## angles
- incoming marketing requests are live priority signals already arriving in chronological order by demand -> rather than retroactively tagging starting from 2009 forwards, process shows in the order they were requested -> keep a shared request log (show title, season year, what was asked, date received) and use that as the processing queue to shrink backlog fastest where it matters most
- full description and making something findable do not need to happen in the same session; characterization accumulates over time -> stub records first (minimal tag like show title + one keyword in EXIF comment, under a minute) transforms an image from invisible to findable, unlocking a second pass -> AI vision tools can then batch-enrich stubbed images with costume color, performer count, scene type, and staging descriptors without hands-on involvement
- a fragment without clean chain of custody cannot be deployed, and an image without clearance cannot be delivered -> treat rights as separate parallel effort independent of visual tagging -> build a one-row-per-production rights ledger (show title, year, contract file location, clearance status) from contract files and production records before next photo session, so any future search result can return clearance status in seconds rather than requiring contract archaeology
