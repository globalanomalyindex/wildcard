## angles
- Ecological false-positive budgets: motion sensors fire on wind/birds/shadows; wildlife researchers partition alerts by "detection specificity" (critical vs. ambient) with different response rules -> instead of raising thresholds uniformly, separate alerts that sit on regulated payment paths (high-specificity) from those in ambient tier, using alert-walk exercise to give compliance an audit trail
- Anesthesia alarm fatigue: ICU bedside alarms were 90% irrelevant, habituation caused near-misses; intervention was positive predictive value (PPV) scoring displayed with alerts -> compute for each alert type the ratio of pages fired to pages requiring action, surface that score to reframe conversation from overload to whether the alert is worth keeping
- Manufacturing two-alarm principle: process alarms (trend warning) vs. safety instrumented system alarms (regulated hard limit) are wired, logged, and audited separately -> treat compliance requirement as SIS constraint distinct from process CPU/disk alerts; separate them into different PagerDuty services, escalation policies, and on-call rotations without changing thresholds, giving compliance a cleaner audit surface
- Garbage collection "stop-the-world" as batching frame: JVM knows certain metric excursions are expected during full GC; alerting layer suppresses or reclassifies evaluation in that window -> if infrastructure emits structured events when batch jobs start/end, alert evaluation for specific metrics (per-host CPU, disk throughput) can be automatically suppressed during that window, bypassing the threshold question for the noisiest class
