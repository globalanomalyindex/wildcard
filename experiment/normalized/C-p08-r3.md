## angles
- screening/confirmation conflation in disease epidemiology -> alert system combines detection and action into one path -> separate high-sensitivity screening alerts (queue/dashboard) from high-specificity confirmation alerts (human page); reserve the regulated compliance paths for a second confirmation condition or a separate channel
- caution vs. warning distinction in aviation cockpits -> alert habituation parallels pilots ignoring real warnings in cockpits full of amber lights -> implement a two-tier notification system (noisy batch-job alerts as caution/visible board, compliance-mapped payment-flow alerts as warning/phone), preserving compliance by reserving the warning tier for regulated paths without changing thresholds
- search image formation in animal behavior -> team lacks a shared mental template for "a page that requires action" -> extract patterns from postmortem logs of actionable pages (time of day, co-occurring signals, specific hosts, payment flows) and document as a one-page decision tree in the runbook to enable faster recognition and build institutional knowledge
- tampering concept from Deming's statistical process control -> alert system fires on predictable, scheduled, normal-range batch-job CPU variation instead of separating common cause from special cause variation -> map alert calendar against batch-job schedule and apply time-windowed, host-scoped suppressions during known batch windows, leaving compliance-mapped paths untouched while generating a concrete artifact (batch-job/alert overlap map)
