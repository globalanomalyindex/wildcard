## angles
- Music pedagogy's "wrong instrument" concept (quit is diagnostic, not terminal) -> exit surveys showing "felt tricked" indicate trial is wrong instrument for customer's mental model -> restructure trial to show live dashboard translating host count to requests/cost projections: "At your current traffic, you'd need X hosts at $Y/mo" (trial becomes tutor not trap)
- Cartography's problem of projection distortion (every projection is accurate and also distorts something) -> per-host pricing accurately describes infrastructure costs but distorts customer's cost-to-value picture -> add pricing explainer modal teaching "1 host typically handles ~50K requests/day" as conversion key without changing unit (teaches rather than estimates)
- Narrative structure's "threshold guardian" pattern (guardian blocks hero's path to test readiness, should appear after hero glimpses what's beyond, not before) -> host limit appearing in week one stops forward progress before users discover value and "felt tricked" churn results -> restructure trial so host limit doesn't appear until day 8-10 after full monitoring cycle and actionable alert received (timing change, not limit change)
- Toll road pricing vs. park admission (toll roads charge per use, parks charge per entry regardless of use volume) -> customers trying to buy park pass but offered toll booth; churn is mismatch in commitment model not unit -> offer flat entry trial that doesn't count hosts but caps total monitored events (keeps finance alignment post-trial, removes week-one gotcha, reframes conversion as "you used X capacity, here's the tier" instead of "you exceeded limit")
