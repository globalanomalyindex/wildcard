## angles
- Contention from independent periodic cycles arriving simultaneously -> desynchronization as structural problem, not throttling problem -> negotiate staggered export-start windows (10-15 minute offsets) to phase-shift cycles and distribute ramp load
- Scheduled periodic load as predictable, clockwork pattern -> periodicity as actionable information for pre-admission -> distinguish "inside known burst window" from "outside it" in SLA design and pre-queue export jobs into rolling smoothing windows
- Hard concurrency ceiling with no differentiated allocation across query types -> saturation as resource-allocation problem -> partition concurrency budget coarsely by query tag ("export-class" vs. "interactive-class") to protect interactive lane without requiring per-tenant attribution
- Fine-grained tenant attribution currently broken, blocking upstream probe -> coarser upstream signal available at API gateway (query queue depth before ClickHouse) -> admission-control gate triggered by queue depth threshold to shed load before cluster saturation, decoupled from attribution instrumentation work
