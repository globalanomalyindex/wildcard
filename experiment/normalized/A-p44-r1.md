## angles
- Full requalification when only partial is needed is the recurring waste pattern -> Your restart loop requalifies from bass clef decoding and pulse-holding onward, but beginner books start earlier; you need a requalification entry point rather than initial qualification resources -> Name where your validated territory ends so each session can ask whether it starts inside the unvalidated zone.
- Two failures co-occurring in a piece look coupled but are entirely separate mechanisms -> Bass clef reading and syncopation both fail in difficult passages but are independent (symbol-mapping vs pulse-stability); treating them as separate variables rather than one compound problem stops the loop -> Structure workstreams to decouple and validate each in isolation, which is faster and clarifies what actually fixed what.
- Short windows fail not from time scarcity but from arriving without a pre-designed protocol -> A 20-minute session aimed at one isolated deficit can complete meaningful validation if the protocol was designed before the session started; the constraint is whether execution begins with a pre-written plan -> Design the "what am I validating and how will I know if it worked" work outside the production window, so session time is spent executing rather than authoring on the fly.
