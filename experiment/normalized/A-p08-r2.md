## angles
- Treating without sorting is wasting scarce resources on proximity not criticality; the field needs walking/tagging before threshold work → alerts lack pathway labels (regulated-flow guardian: yes/no) so you can't route selectively → attach pathway tags to each alert in a two-hour labeling exercise before tuning anything else
- In undifferentiated alert piles, salience is anti-correlated with criticality: high-frequency noise (CPU/disk on batch jobs) trains responders to expect noise, making genuine signals feel like more noise → create lane separation before the page lands to structurally interrupt cognitive pull toward high-frequency alerts
- Ad-hoc individual suppression (engineers muting phones) recreates the unlabeled-expectant failure: if muted alert fires on genuine regulated-path failure, no one knows the page didn't land → convert informal muting into institutional suppression criteria (zero confirmed true positives, batch-job correlation confirmed, doesn't touch tagged regulated paths) with automatic re-triage trigger, making it auditable and reversible
