## angles
- Synchronous export jobs create constructive interference in the time domain, concentrating load rather than distributing it -> treating load as a phase synchronization problem rather than a volume problem -> phase-offset each tenant's export job to spread execution across the hour window (e.g., stagger by :00, :05, :10, :15) to destructively interfere with the spike while preserving contract terms and requiring no new infrastructure
- The top-of-hour spike is a high-frequency artifact in the time domain -> a temporal low-pass filter can smooth it by decoupling job submission from execution -> front the export queue with a dispatch buffer that spreads accumulated queries across a window (e.g., submit at :00 but issue ClickHouse jobs over 15–30 minutes) to trade the sharp spike for a gentler slope
- Tenant IDs are threaded inconsistently and full metering is not in place -> temporal pattern analysis (phase localization in signal processing) can triangulate a signal source without explicit tagging -> the synchronized nature of the spike immediately identifies the cohort of tenants with :00-scheduled exports as a coarse filter, narrowing the population for manual inspection before full attribution plumbing is built
