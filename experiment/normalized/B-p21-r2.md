## angles
- Archive content is complete but lacks time-indexed access layer; local speech-to-text tools like Whisper can generate word-level timestamps without uploading, turning transcript generation into a batch processing problem rather than a storage problem -> approach transcript generation as cutting an access mask, not recovering lost data -> run Whisper batch overnight on one drive to generate timed text layer over all files within local constraints.
- Guest retrieval pressure (fuzzy, topic-level recall) and publisher pressure (exact wording with timestamps) require different transcription resolutions -> not all 410 episodes need uniform precision -> identify publisher's specific episode list first and apply coarse processing (keyword spotting, speaker-change detection) broadly while reserving fine-grained transcription for the alignment-marked subset.
- Two drives with inconsistent naming (date-based vs. guest-based) prevent accurate file retrieval in any downstream pipeline -> folder naming is a mask-alignment failure -> cut a master manifest first: map every file to common fields (guest name, episode number, date, duration, drive path) in a registration step, then all retrieval operations become lookups against a known coordinate system.
