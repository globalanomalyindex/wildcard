## angles
- Epidemiologists build contact graphs before incidents so exposure scope is knowable in advance, without quarantining everyone -> rotation chaos comes from missing a dependency graph linking credentials to services and people that hold them -> draw a credential contact map (each secret as a node, each service/engineer/CI system as an edge) before touching tooling to create the triage list
- Buildings design compartmentalization so one burning room cannot cascade, using structural control rather than behavioral policing -> engineer friction comes from asking people to change behavior, but compartmentalization stops credentials crossing zone boundaries by construction -> give each environment (Heroku, AWS, legacy VPS) its own short-lived, zone-scoped credentials so a compromised dev laptop has blast radius at most one zone
- Rental companies photograph every scratch before handoff because disputes are otherwise unresolvable; the inspection sheet is a trust protocol -> hardcoded credentials in history is partly pre-commit hygiene but fundamentally a "no inspection before transfer" problem -> reframe scanning as an inspection checklist at PR open (one required checkbox, five seconds, audit trail for SOC 2) that gives engineers agency rather than policing them
- Theater companies run understudy rehearsals on purpose under controlled conditions so substitution becomes boring -> rotation procedure was theoretically documented but never load-tested -> run one deliberate rotation rehearsal for a critical non-production credential on a chosen Thursday afternoon to discover which services don't honor the new key, which engineers panic, and which parts of the runbook are wrong
