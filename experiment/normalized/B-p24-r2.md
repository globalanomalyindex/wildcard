## angles
- Temporal path reachability: 30% of viewers drop before reaching the build footage, not because the content is missing but because the path to it fires in the wrong order and takes too long -> shorten the temporal path to high-value content by showing a visually arresting fragment of the build within the first 10-15 seconds as live footage, not as a caption
- Editing bottleneck: 4-6 hours of footage processing includes high-slack work (reviewing unused footage, color-matching brief B-roll, fine-trimming inconsequential cuts) that doesn't affect output quality driving retention -> identify and either eliminate, defer, or edit slack segments to a looser standard to compress total edit time without touching critical nodes (hook, key build sequences, audio clarity)
- Impression count and retention are two disconnected subgraphs with different upstream parents; impression count is downstream of thumbnail/title/topic/algorithm while retention is downstream of hook/pacing/expectation matching -> hold topic and build type fixed while systematically varying one of thumbnail vs. title vs. hook across several videos, and use the shared parent test (if all three metrics move together, look for the upstream node feeding all of them before testing leaf nodes individually)
