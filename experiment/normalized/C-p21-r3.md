## angles
- Broadcast captioning uses batch speech-to-text on audio to generate rough subtitle files (SRT) as searchable timestamped word lists rather than polished transcripts, lowering accuracy requirements; 70-80% accuracy enables finding quotes in seconds rather than scrubbing entire episodes -> locator files are a lighter deliverable than full transcripts for the publisher's need -> run Whisper offline in batch mode and treat output as timestamp map rather than finished document

- University archives use minimal-level processing ("More Product, Less Process") to cover everything with rough finding aids rather than perfect catalogs covering only part of the collection; rough metadata (guest, date, summary, one pull-quote) built from episode boundaries and show notes surfaces ~80% of needed quotes -> instead of transcribing all 410 episodes, create one-sentence episode summary with notable quote from audio endpoints and available materials -> build a sparse index in a spreadsheet before touching audio

- File recovery treats folder structure as irrelevant, indexing by content fingerprint (audio duration, creation timestamp) rather than path; this solves the inconsistent naming problem across two drives by ignoring it -> run recursive ingest across all drives without navigating folder structure, using ffprobe to extract duration and file-modified date -> create a ground-truth manifest of all .mp3/.wav files before processing

- Oral history projects build a highlights-reel manifest before full transcription by listening at 2-3x speed and logging only quotable segments with timecodes; the manifest is the actual deliverable for licensed quotes -> identify 30-40 episodes most likely to contain licensable quotes (prominent guests, high-download episodes, remembered strong exchanges) and apply focused human review at 2x speed -> ask the publisher for search terms to pre-filter which episodes to process rather than working chronologically
