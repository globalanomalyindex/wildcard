## angles
- Fragmented temporal record across five substrates with no correlation -> loss of sequence readability is not a data loss problem but a retrieval-order problem -> think of recipes by date-cooked or date-saved as an event horizon that correlates across all five substrates simultaneously, reconstructing the column
- Full pore networks per substrate with zero connectivity between them -> Sunday planning is the well but retrieval pressure drops because nothing flows across substrate barriers -> diagnose which barriers are structural (disconnected) versus diagenetic (actively decaying) before choosing a fix, as the answer depends on the kind of barrier
- Each recipe carries visible provenance (grandmother's card, Instagram screenshot, bookmark) already present but never indexed -> add minimal capture metadata at save-time rather than attempt reconstruction later -> a small fixed set of fields (source, date saved, date cooked, household reaction) logged before burial obscures the information
- Dead links and paywalled pages overwrite the original record post-capture -> pointers to original locations are vulnerable to diagenesis while copies are primary preservation -> distinguish between a pointer (vulnerable to later erosion) and a copy (rock cast of the grain before burial) in capture habit
