## angles
- Surface readings decouple from true state (vent temperature ≠ reservoir temperature; "available" flag ≠ physical readiness) -> structured condition capture at the moment of return encodes actual state at origin and survives intact -> consistent check-in photos of three or four key fields restore the scrambled signal and let you answer disputes without memory contests
- Discharge readings are meaningless without locked recharge baselines (output chemistry requires knowing input signature; return damage claims require dispatch condition records) -> the unit of measurement is the pair, not either event alone -> standardize dispatch records with same structure as return records so each unit carries its own baseline through the rental
- High-volume surges are hard to manage during the pulse (Saturday 7–9 am is like a geyser eruption) -> the practical move is to measure pre-pulse state, not the surge itself (Friday-afternoon queue-depth audit tells you lead time) -> flag which units must clear tonight, which slot to Monday, who needs calls before Saturday morning
- Stale readings are credible readings with high residence time and need a discount applied (excavators without inspection for weeks show weeks-old status, not current state) -> make residence time visible in the system so age becomes an explicit credibility weight -> units with high residence time in limbo should be flaggable in a sort, not discovered by walking the yard
