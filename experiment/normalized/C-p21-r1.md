## angles
- Whisper (and hosted wrappers) runs locally on batches unattended -> transcription can happen automatically on external drives using a shell script -> prioritize running Whisper only on episodes with quotable material rather than all 410, waking to timestamped transcripts
- Transcripts are plain text (60-80 KB per hour) while hosting caps uploads -> decouple transcript archive from audio delivery using a separate static site or database -> hosting transcripts on Netlify, Notion, or GitHub avoids touching the audio hosting layer
- Libraries solved this at scale by separating finding aids from items and staffing cataloging as a retrospective push -> treat the 410-episode backlog as a defined archival project, not ongoing maintenance -> parallel effort on high-priority episodes creates a usable index before the publisher's deadline
- Court reporters separate verbatim capture and time-indexing from curation -> raw Whisper output with timestamps is searchable without polishing -> give the publisher raw transcript plus audio timestamp so they verify their few needed quotes rather than requiring a curated full archive
