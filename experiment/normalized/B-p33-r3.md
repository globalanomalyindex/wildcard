## angles
- Fragility accumulates invisibly (poor routing, untracked inventory, high first-visit failure rate) before summer arrives -> structural mechanics of slope failure (stress loads gradually, then minor trigger cascades) -> intervene before peak season by discharging fragility (pre-season drainage equivalent): improve routing efficiency, track inventory, reduce first-visit failures before June so summer demand lands on a stable system.
- Best installer exposed to same inefficient cross-city routing as everyone else, capacity erodes first when dispatching treats all nodes equally -> heterogeneous terrain fails first at steepest slope with weakest substrate, cascading to rest of system -> selectively reinforce highest-leverage node's time protection; ask whether dispatch has mechanism to preferentially protect best performer or treats all nine as equivalent.
- 25% second-visit rate is a failure reading with no early-warning signal because no inventory tracking -> creep rate (not absolute displacement) is what reveals approach to failure; slope moving 8mm this month vs 2mm last month signals instability -> instrument the system: per-van, per-part count updated at job close to track whether van's capacity gap is improving or worsening month-over-month, giving early-warning window before cascade.
- Once summer queue builds, system does not return to previous state when call volume drops -> slope memory: water still in soil, micro-fractures still open, system remembers loaded state -> reducing calls by 10% doesn't discharge backlog because system state has hysteresis; queue drains only when the stored state (not just flow) changes, requiring drainage of pore pressure equivalent not just stopping the rain.
