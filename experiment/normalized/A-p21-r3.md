## angles
- Archival restoration requires triage before treatment, not uniform application of effort to every asset -> Running transcription on all 410 episodes in three weeks is infeasible, but identifying the high-value cohort first and transcribing only those shrinks the scope to manageable -> Union the publisher's quote needs with guest inquiries to define a smaller target set for transcription, then work highest-value to lowest
- Restoration and finding aids are separate artifacts that never modify the source -> Automated transcription tools generate sidecars (timestamped JSON/SRT files) alongside source audio without touching the original -> Start generating transcripts now using local tools like Whisper before reorganizing folder structure; sidecar approach means file naming and folder chaos don't block the work
- Overcorrection in restoration removes real content and creates the illusion of completeness -> Using per-word confidence scores to flag uncertain regions preserves integrity rather than silently introducing errors -> Deliver two tiers to the publisher: verified-to-threshold quotes and regions flagged for human review, rather than one undifferentiated transcript
- Finding aids built during restoration create permanent random-access infrastructure out of a sequential-access archive -> The three-week deadline forces exactly this investment, making future guest inquiries and licensing requests costless -> The transcription work serves two simultaneous clients: the publisher's immediate deadline and the archive's long-term operation as retrieval infrastructure
