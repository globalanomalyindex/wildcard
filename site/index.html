wildcard

skill for claude code install

wildcard is an experimental skill for claude code that, at random, introduces an unrelated expert to your current task or project. in doing so, the token pool is seeded with unorthodox directions for standard chain-of-thought systems to work with - influencing more creative thinking, problem-solving, and idea generation in large language models.

this project is an experiment in introducing and emulating non-linear creative thinking in artificial intelligence systems, without straying away from current artificial intelligence technology and capabilities.

a synthetic default-mode network

An LLM's chain of thought is task-locked. It never wanders, so the "pop-in" of a remote idea can't happen; the architecture forbids it. wildcard externalizes the missing faculty: it injects the remote associate the on-task reasoner would never wander to. Different conditioning, not higher temperature - structured divergence, never noise.

how it works

  1. detect - distill the structural sketch of your project
  2. draw - real entropy picks one of 344 niche disciplines, outside the model
  3. specialize - the draw grows into a hyper-specific practitioner
  4. notice - only genuine relational matches; honest abstention allowed
  5. seed - 2 to 4 optional provocations, yours to plant or discard

map relations, not nouns

A bad connection shares a noun: "your code has cells, I study cells!" A real one maps structure: your retry backoff and a predator-prey cycle are the same oscillation, and ecologists found stochastic jitter stops populations synchronizing into a crash. Same relations, different domain. That bar makes "never lies for a connection" checkable.

looking to nature

  • cross-pollination - the carrier doesn't choose what it carries
  • germination - most seeds never sprout; you pick what takes root
  • outbreeding - a distant cross escapes a closed gene pool
  • stochastic resonance - the right noise amplifies a weak signal
