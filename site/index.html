wildcard

skill for claude code install

wildcard is an experimental skill for claude code that, at random, introduces an unrelated expert to your current task or project. in doing so, the token pool is seeded with unorthodox directions for standard chain-of-thought systems to work with — influencing more creative thinking, problem-solving, and idea generation in large language models.

this project is an experiment in introducing and emulating non-linear creative thinking in artificial intelligence systems, without straying away from current artificial intelligence technology and capabilities.

a synthetic default-mode network

An LLM's chain of thought has no default mode. It is relentlessly task-locked — it never wanders, never lets something irrelevant surface, because every token is conditioned on staying on-task. So it can't have the "pop-in"; the architecture forbids it. wildcard externalizes the one faculty the architecture is missing — a synthetic default-mode network that injects the remote associate the on-task reasoner would never wander to.

Not higher temperature, which only flattens the distribution into noise — but different conditioning. A specific expert persona translates the model's high-probability mass to a distant, still-coherent region of latent space. Structured divergence, never randomness.

how it works

  1. detect — distill the structural sketch of your project: its moving parts, flows, and tensions. Structure, not stack — so it works on code, prose, or research alike.
  2. draw — a shell script picks one of 344 niche disciplines with real entropy, outside the model, so no field is quietly favored.
  3. specialize — the coordinate grows into a hyper-specific practitioner with a real toolkit, steered by a drawn lens into a non-obvious corner.
  4. notice — only genuine relational matches surface; honest abstention when little maps.
  5. seed — two to four optional provocations, yours to plant or discard.

map relations, not nouns

A bad connection shares a surface feature — "your code has 'cells,' I study biological cells!" A real one maps relational structure: "your retry backoff and a predator–prey cycle are the same oscillation, and ecologists found stochastic jitter stops the populations from synchronizing into a crash — have you considered jitter in your backoff?" Same relations, different domain. Encoding that bar is how "doesn't lie for the sake of a connection" becomes a checkable property instead of a vibe.

looking to nature for the mechanism

  • cross-pollination — a pollinator carries genes between flowers without choosing which; the draw is that undirected carrier.
  • seed dispersal & germination — most seeds never germinate, and that is the honest hit rate; you decide which provocations take root.
  • outbreeding — a distant cross escapes the inbreeding depression of a closed gene pool, the way a foreign field escapes mode collapse.
  • stochastic resonance — in a nonlinear system, the right amount of noise amplifies a weak signal instead of drowning it.

install

ln -s "$(pwd)" ~/.claude/skills/wildcard

Then run /wildcard in any Claude Code session — and meet your expert.
